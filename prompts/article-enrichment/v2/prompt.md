# HNHOT 文章语义增强提示词 v2

## 任务

逐篇加工《海南日报》有效报道，生成第一次入库所需的事实性摘要和原文标记。这里只描述本篇报道，不判断对象是否值得建立人物页、活动页、项目页或规划页，也不与站内历史对象合并。

## 事实边界

标题和正文是主体名称、身份、事件、规划文件与所有语义判断的唯一依据。不得使用外部知识补全原文没有写出的事实。

`location_candidates` 和 `topic_candidates` 是受控词表：地点和主题只能选择其中已有的 ID。主体和事件没有候选词表，凡原文明示且属于本篇主要信息的对象都可以自由提取。候选为空不代表原文没有主体或事件。

## 输出

返回严格符合 `schema.json` 的一个 JSON 对象：

- 保持输入条目顺序和 `candidate_id` 不变。
- 将输入中的 `schema_version`、`prompt_version` 和 `input_fingerprint` 原样返回，不自行计算或改写。
- `evidence` 不是解释、理由或概括，必须从本条标题或正文中连续逐字复制一段原文。不得同义改写、删词拼接、调整语序、替换标点或引号，也不得把原文中分散的两段文字合并。先定位原文片段，再填写对应字段；找不到可直接复制的证据时，不输出该对象。
- 没有可靠内容时返回空数组，不得用模板内容补齐。
- 不添加网址、来源元数据、评分、推荐理由、长期实体 ID、页面判断或 schema 之外的字段。

## 摘要

`ai_summary` 面向普通读者，原则上只写一句简洁、客观的中文：

- 直接写最值得知道的具体问题、变化、行动或结果。
- 只保留最有信息量的地点、日期、数字、政策影响、项目进展或下一步安排。
- 删除会议流程、礼仪动作、口号、空泛评价、重复背景和不影响实际结果的行政过程。
- 评论文章概括具体问题及文中已经提到的实际应对，不以“文章指出”“文章主张”开头。
- 正文缺失时返回 `null`，绝不能只根据标题推测。

## 范围

范围分类回答“这条信息主要提供哪一种观察视角”，不只看发生地，也不因出现“海南”或“国际”字样就决定分类。

- `mixed`：海南与境外直接发生联系，或核心价值明确面向开放和国际交流。
- `domestic`：海南与中国其他省区市发生直接联系，且跨省联系是核心信息。
- `hainan`：主要关乎海南内部的本地生活、民生治理、产业、科技、规划建设、基础设施、生态文化和公共服务。
- `foreign`：主要事件和行动主体在中国以外，且没有海南主体直接参与。
- `national`：全国性或中国其他地区信息，且没有海南主体直接参与。

从标题或正文连续逐字复制简短的 `scope_evidence`，同样不得改写或拼接。

## 主体

`subjects` 是供后续跨篇归并使用的原文主体记录，不是人物页、机构页或项目页的准入名单。提取本篇中与具体行动、结果或影响直接相关的人物、政府、组织、企业或项目：

- 提取实施、宣布、调研、建设、运营、负责或直接承受核心行动的主体和对象，以及取得明确成绩、奖项、任命、纪录或阶段成果的人物与团队。
- 综述、盘点和名单类报道中，只要每个具名主体都对应一项独立、明确的行动或结果，就应分别提取；不得只挑一个代表人物，也不得因为人物较多而随意截断。
- 新闻记者、普通引语提供者、礼节性出席者、陪同者，以及只在背景表述中出现的人名通常不提取。
- 比赛对手、非核心合作搭档和仅用于比较或衬托的人物，若不是本篇所记录事实的对象，通常不提取。
- 原文明确写出身份或本篇角色时填写 `role`；否则为 `null`。
- `name` 保留原文中的规范名称，不添加头衔，不生成主体 ID。
- 原文使用“以下简称”“简称”“下称”“又称”或“俗称”等明确声明主体简称或别称时，在该主体的 `aliases` 中逐项记录。每项 `name` 必须逐字保留原文中的简称或别称，`evidence` 必须连续复制同时包含规范名称、声明用语和简称或别称的原文片段。
- 不得根据名称长度、惯用说法或上下文自行发明简称；原文没有明确声明时省略 `aliases`。同一个简称在该主体中只保留一次。
- 同一主体在一篇文章中只保留一次。
- 不判断该主体以后是否还会出现，也不判断是否值得建立独立页面；这些决定属于后续跨篇归一化和页面准入阶段。

## 地点和主题

- `location_mentions` 只能从本篇 `location_candidates` 中选择；一篇报道涉及多个海南行政区时可全部保留，不因数量较多而只取前三个。
- `topic_mentions` 只能从 `topic_candidates` 中选择。
- 每个选择都必须有对应的原文证据。

## 事件

`events` 自由提取原文明示的命名活动或本篇核心具体行动，可以有多项：

- 包括会议、论坛、展览、节庆、赛事、公共活动、事故灾害，以及“某人调研某项目”等可明确概括的核心行动。
- 对命名活动优先保留原文中的完整届次或年份，例如“2026世界人工智能大会暨人工智能全球治理高级别会议”。
- 对具体行动使用简洁的“主体 + 动作 + 对象”名称，例如“刘小明调研环岛旅游公路”。
- 不判断 `existing` 或 `new`，不生成事件 ID，不推测所属长期活动系列。
- 普通评论、背景材料或无法形成明确行动的内容返回空数组。

## 规划文件

`plans` 只标记正文或标题中使用书名号 `《》` 明确写出的规划、纲要、方案、行动计划或其他具有规划安排性质的政策文件：

- `name` 必须完整保留书名号和其中的原文标题，例如 `《海南省旅游公路发展规划》`。
- 文件必须在上下文中具有规划、部署或行动安排性质；文学作品、影视作品、普通书籍、法律法规和仅被引用的新闻标题不属于此字段。
- 同一文件在一篇文章中只保留一次。
- 不生成规划 ID，不补充原文没有写出的发布机关、发布日期、效力层级或文件内容。
- 没有符合条件的规划文件时返回空数组。
