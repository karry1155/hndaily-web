# AIHOT 2.0 设计系统分析

## 快照口径

- 采集时间：2026-07-16（America/Los_Angeles）。
- 页面业务日期：2026-07-17（北京时间）。
- 版本证据：更新日志称 2026-07-12 完成 AIHOT 2.0 全站重做；公开 API 于 2026-07-15 报告 `apiVersion 1.4.0`。

## 总体原则

AIHOT 2.0 使用「墨青编辑部」视觉系统。桌面端是固定信息侧栏加宽时间线，手机端是紧凑阅读流加四项底部导航。两端共享品牌、颜色、文字和状态语义，但不强行共享同一种卡片密度。

## 响应式层级

| 范围 | 当前用途 |
|---|---|
| `> 960px` | `180px` 桌面侧栏、宽时间线卡片、桌面搜索与主题切换 |
| `<= 960px` | 隐藏桌面侧栏，启用四项固定底栏，页面进入移动信息结构 |
| `<= 640px` | 页面 gutter 从 `16px` 收紧到 `12px`，标题和时间列进一步压缩 |

关键变量来自当前公开 CSS：

```css
--bp-md: 640px;
--bp-lg: 960px;
--nav-width: 180px;
--m-tabbar-h: 54px;
--m-gutter: 18px;
--touch-target: 44px;
--container-feed: 720px;
--container-detail: 640px;
```

## 颜色与主题

- 深色页面基底：`#10151c`。
- 浅色页面基底：`#f4f5f6`。
- 浅色品牌墨青：`#135e6b`；深色使用更亮的青色角色。
- 排名依次使用砖红、暖橙、琥珀；不再使用高饱和霓虹渐变。
- 主题模式为 `dark | auto | light`，由根节点 `data-theme` 和 `data-theme-mode` 驱动。
- 首次访问跟随系统；手动选择后由前端状态层保留。

## 字体与文字层级

- 统一使用系统无衬线字体栈，中文优先 PingFang SC、Hiragino Sans GB、HarmonyOS Sans SC 和 Microsoft YaHei。
- 基础字号 `14px`，正文行高约 `1.5–1.75`。
- 手机新闻标题约 `16–18px`，摘要约 `14px`，来源/时间/分数约 `11–12px`。
- 桌面时间线标题约 `15–17px`；摘要和推荐理由通过颜色与分隔线降级。
- 数字、时间、章节编号和眉题使用等宽字体角色。

## 表面与密度

- 桌面：热点和每条时间线新闻都是白色/深色卡片，卡片间保留明确间距。
- 手机：首页热点保留强卡片；普通时间线退回连续阅读流，只用分隔线。
- 手机详情页减少装饰，AI 摘要使用可折叠容器，正文保持长文阅读宽度。
- 手机日报保留「今日看点」摘要卡片，正文按章节连续排列。
- `12px`、`16px` 为主要圆角；触控目标以 `44px` 为基准。

## 交互与可访问性

- 键盘焦点：品牌色 `2px` 外轮廓并带 `2px` 偏移。
- 手机链接、按钮、`summary` 和 radio 都启用 `touch-action: manipulation`。
- 底栏为页面根级组件，详情页隐藏；其它主要页面根据路径高亮精选、全部、日报或更多。
- 主题控件使用 `radiogroup` / `radio` 语义；收藏使用 `aria-pressed`。
- 正文为固定底栏预留底部空间，并兼容安全区。

## 与旧快照冲突的结论

- 旧文档所述“桌面新闻流不使用逐条卡片”已过时：AIHOT 2.0 桌面端明显使用卡片时间线。
- 手机端仍保持开放列表，因此不能把桌面卡片规则机械复制到手机。
- 旧截图只有首页；当前资料已补齐搜索、收藏、详情、日报、主题、更多/外观、更新日志和 Agent 接入。
