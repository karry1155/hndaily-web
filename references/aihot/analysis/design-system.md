# AIHOT 设计系统分析

## 总体原则

AIHOT 的核心不是“小字号”，而是让容器、间距、文字和操作形成稳定的密度梯度：页面只保留一个强容器（热点），分类作为开放工具栏，新闻作为连续阅读流。视觉重量从热点卡片逐步降低到新闻分隔线。

## 响应式层级

| 范围 | 观察到的用途 |
|---|---|
| `> 960px` | 桌面侧栏、桌面工具区和更宽内容布局 |
| `<= 960px` | 隐藏侧栏，启用移动顶栏与固定底栏；正文预留底栏安全空间 |
| `<= 640px` | 收窄页面边距、时间轴列宽、卡片内边距和标题尺寸 |

不要为每个组件发明独立断点。HN·HOT 应统一使用现有 `760px` 移动断点，并把 AIHOT 的两层思想映射为“移动端基础”和“特别窄屏微调”。

## 字体

- 基础字体：系统无衬线字体栈，中文优先 PingFang SC、HarmonyOS Sans SC、Microsoft YaHei。
- 基础字号约 `14px`，基础行高约 `1.5`。
- 新闻标题约 `14–15.5px`，字重 `600–700`，行高约 `1.5–1.55`。
- 摘要约 `13.5–14px`，行高约 `1.65`，通常限制两行；窄屏复杂卡片可放宽三行。
- 元信息约 `11–12.5px`，使用次级颜色和等宽数字特性。
- 章节标题约 `15–17px`，不与页面品牌或主标题竞争。

## 间距

可复用间距阶梯：

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
```

- 移动页面 gutter：`12–18px`。
- 普通行纵向内边距：约 `8–13px`。
- 模块间距：约 `12–20px`。
- 热点标题区：约 `10–16px`。
- 热点行：约 `8px 12–16px`。

## 表面和边界

- 页面背景是低对比浅灰；内容主体多为透明或同背景。
- 热点卡片：白色表面、`1px` 边框、约 `12px` 圆角、极浅阴影。
- 新闻列表：不用逐条卡片，使用浅色 `1px` 分隔线。
- 分类：每个胶囊独立拥有背景/边框；分类集合没有大外框。
- 阴影只用于抬升少数高优先级模块，不用于每条新闻。

## 颜色角色

不要照搬具体色值，复用角色：

```css
--surface-page: /* 页面背景 */;
--surface-card: /* 热点等强容器 */;
--text-primary: /* 标题 */;
--text-secondary: /* 摘要 */;
--text-tertiary: /* 时间和来源 */;
--border-soft: /* 新闻分隔 */;
--border-card: /* 热点边框 */;
--brand: /* 当前分类、当前底栏 */;
--rank-1: /* 红 */;
--rank-2: /* 橙 */;
--rank-3: /* 青 */;
--rank-rest: /* 蓝灰 */;
```

HN·HOT 应继续使用自己的品牌蓝和现有深浅主题变量，只映射角色，不复制 AIHOT 品牌色。

## 触控与动画

- 底栏、收藏按钮等触控目标保持约 `40–44px`，图标视觉尺寸约 `16–20px`。
- 触屏使用 `touch-action: manipulation`，弱化点击高亮。
- 动画约 `120–220ms`，只处理颜色、边界、阴影和轻微位移。
- 键盘焦点使用品牌色 `2px` 外轮廓。
