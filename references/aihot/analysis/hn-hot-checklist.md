# HN·HOT 对标 AIHOT 2.0 检查表

## 响应式外壳

- [ ] 桌面侧栏与内容区职责分离，侧栏不随长列表滚动。
- [ ] `<= 960px` 切换为精选、全部、日报、更多四项底栏。
- [ ] 详情页隐藏底栏；普通页面为 `54px` 底栏和安全区预留空间。
- [ ] `393 × 852` 下没有横向溢出或被底栏遮挡的最后一行。

## 热点与时间线

- [ ] 热点显示排名、代表标题和独立信源数。
- [ ] 热点不足时允许隐藏，不用空卡片占位。
- [ ] 桌面新闻使用卡片时间线；手机新闻使用开放列表。
- [ ] 分数、精选、重复报道和推荐理由各自有稳定位置。
- [ ] 重复报道折叠到代表事件，不因公司名相同就错误合并。

## 搜索与分类

- [ ] 搜索使用服务端 `q`，不只在当前页本地过滤。
- [ ] 来源和类型筛选可组合，并保留有效搜索参数。
- [ ] 手机类型菜单可折叠，选中值在关闭状态仍可见。
- [ ] 搜索结果展示总数、取消入口和明确的空状态。

## 本地状态

- [ ] 匿名收藏明确说明只存当前浏览器。
- [ ] 收藏按钮同步 `aria-pressed` 和可见状态。
- [ ] 已读、收藏、主题与更新日志已读使用不同键名和事件。
- [ ] 存储损坏时回退为空状态，不阻塞页面渲染。

## 内容详情

- [ ] AI 摘要可展开/收起且使用原生语义。
- [ ] 正文提取不能确认时降级到摘要 + 原文链接。
- [ ] 原文、分享、收藏、Markdown 导出在手机端可触达。
- [ ] 标题、数字、时间、链接、代码和表格在正文中得到保留。

## 报告与主题

- [ ] 日/周/月切换不会丢失当前报告语义。
- [ ] 日报保留今日看点、阅读时长、章节和快讯结构。
- [ ] 主题卡片使用稳定 slug，并展示清晰范围与精选数量。
- [ ] 主题由标签聚合时，仍有人工可控的展示名和说明。

## 截图验收

- [ ] `1440 × 1000`：侧栏、搜索、热点、时间线首屏完整。
- [ ] `393 × 852`：热点、筛选、新闻、底栏密度正确。
- [ ] 搜索、收藏、详情摘要、日报、主题和更多页都有独立截图。
- [ ] 每张截图都已以原始分辨率检查，排除空白和加载中状态。
