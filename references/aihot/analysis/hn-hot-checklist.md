# HN·HOT 对标检查表

未来提出“向 AIHOT 靠拢”时，按以下顺序检查，不要只改字体大小。

## 信息架构

- [ ] 移动顶部只显示品牌与日期/更新时间。
- [ ] 四项主导航固定在底部，并支持安全区域。
- [ ] 正文底部补偿与底栏实际高度一致。
- [ ] 热点、分类、日期新闻流顺序符合当前产品要求。

## 容器层级

- [ ] 首屏最多一个强卡片容器，通常是“时下要闻”。
- [ ] 分类区域直接连接页面背景，没有总胶囊外框。
- [ ] 日期新闻流没有外层大卡片。
- [ ] 普通新闻条目只用浅分隔线。
- [ ] 阴影没有重复出现在每条新闻上。

## 密度

- [ ] 移动页面左右边距约 `12–18px`。
- [ ] 热点行纵向内边距约 `8px`。
- [ ] 新闻行纵向内边距约 `10–13px`。
- [ ] 分类按钮高度约 `30–36px`。
- [ ] 首屏能看到热点、分类以及至少一至两条新闻。

## 文字层级

- [ ] 模块标题不超过页面主标题权重。
- [ ] 新闻标题约 `14–16px`，摘要约 `12.5–14px`。
- [ ] 来源和时间使用次级颜色与更小字号。
- [ ] 热点标题最多两行，新闻摘要最多三行。
- [ ] 完整文本仍留在 DOM 中，截断只由 CSS 完成。

## 操作

- [ ] 收藏按钮靠右且不压缩正文。
- [ ] 图标视觉尺寸小，但触控面积不小于约 `40px`。
- [ ] 当前分类和当前底栏项使用品牌色高亮。
- [ ] 横向分类可以触摸滚动且不显示滚动条。
- [ ] 键盘焦点清晰。

## 浏览器截图验收

至少验证以下视口：

- [ ] `393 × 852`：常见 iPhone 宽度。
- [ ] `733 × 860`：宽移动/窄平板，确保仍走移动布局。
- [ ] `1440 × 1000`：桌面端无回归。

每次截图同时记录：顶部可见内容、热点高度、分类是否开放、首屏新闻数量、底栏矩形位置、是否出现横向溢出。

## 常见误区

- 只缩字号，没有移除嵌套容器。
- 固定底栏嵌在含 `backdrop-filter` 或 `transform` 的祖先内。
- 修改了源 CSS，却没有重建实际预览/发布目录。
- 测试只搜索关键词，没有检查最终级联块或浏览器计算样式。
- 参考图要求“开放列表”，实现却继续给日期组添加圆角和阴影。
