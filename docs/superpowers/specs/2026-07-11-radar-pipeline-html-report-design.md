# HN·HOT Radar 数据加工 HTML 报告设计

## 目标

生成一份可独立打开的完整 HTML 报告，准确解释海南日报数据从 `hndaily-skill/crawler.py` 抓取开始，经过确定性脚本、模型语义分析、脚本评分与筛选、公开 JSON 持久化、索引生成，最终渲染为 HN·HOT 首页的全过程。

报告主线只覆盖当前“精选 / 时下要闻”使用的 Radar 流水线；AI 日报作为附录说明差异。报告同时指出“推荐理由”和滚动加载应该接入的具体层次，但本阶段不实现这两项产品功能。

## 输出

- 独立报告：`docs/hndaily-radar-pipeline-report.html`
- 可在浏览器中直接打开，无需服务器、网络或外部资源。
- HTML 内嵌样式、交互脚本和精简示例数据。
- 报告不是生产页面，不进入站点构建输出。

## 报告结构

1. 概览：从抓取到页面的完整阶段图。
2. 阶段导航：点击阶段切换当前输入、处理、输出和责任边界。
3. 文件与函数地图：每个阶段对应真实文件和关键入口。
4. JSON 演进：展示各阶段实际字段结构及新增、删除、私有、公开字段。
5. 模型边界：明确哪些字段交给模型、哪些字段禁止由模型生成、模型输出如何校验。
6. 脚本边界：预过滤、稳定 ID、基础分、最终分、去重、阈值、时间衰减和原子发布。
7. 页面数据：精选库、完整读报层、focus 索引、分类/日期/搜索索引与 HTML 的关系。
8. 推荐理由缺口：提出模型字段、验证契约、公开 JSON 和页面渲染的建议落点。
9. 滚动加载建议：说明沿用分页索引并由前端逐页加载的方案，避免一次发布巨大列表。
10. AI 日报附录：简述 Daily Digest 与 Radar 的输入、模型契约和输出差异。

## 视觉编码

- 蓝色：确定性脚本阶段。
- 紫色：模型语义分析阶段。
- 绿色：公开发布 JSON 或页面输出。
- 灰色：内部中间文件、审计和失败保护。
- 箭头只表达真实的数据依赖，不把调用顺序与文件复制混为一谈。

## 交互

- 阶段按钮支持鼠标与键盘，切换时只更新一个详情区域。
- JSON 示例可在“输入 / 输出”之间切换。
- 所有核心信息在无 JavaScript 时仍以文本摘要可读。
- 不增加与理解数据链无关的筛选器、搜索或装饰性指标卡。

## 取证要求

- 从实际脚本、测试、数据契约和 `content/` 示例中提取信息。
- 不根据文件名推断函数职责；关键字段必须能追溯到代码或契约。
- 示例 JSON 应精简但字段名真实，不包含凭空创造的现有字段。
- 对计划新增的 `recommendation_reason` 使用“建议字段”标记，不能误称为当前实现。

## 验收

- 报告覆盖 crawler → 页面全部关键阶段。
- 模型和脚本职责可一眼区分。
- 至少展示原始 issue、source candidate、模型输入、模型输出、stored item、focus index、date/category index 和页面摘要的 JSON 结构。
- 报告明确解释当前分页、未来滚动加载以及推荐理由的数据设计。
- 浏览器桌面和移动视口无横向溢出、文字重叠或失效交互。
- 报告中的本地文件引用与实际仓库一致。
