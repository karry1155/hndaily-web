# HN·HOT 移动导航与精选信息层级设计

## 目标

窄屏参考 AIHOT 的移动信息层级：顶部仅显示品牌与更新时间，当下重点紧随其后，主导航固定在屏幕底部。宽屏继续沿用左侧导航，并保留搜索。

## 响应式规则

- `<= 760px`：顶部显示 HN·HOT 与“更新至 YYYY-MM-DD”；隐藏菜单按钮与搜索框。
- `<= 760px`：正文顺序为当下重点、分类标签、日期精选列表。
- `<= 760px`：底部固定四项导航：精选、全部信息、AI 日报、收藏。
- `> 760px`：保留左侧导航、分类与搜索并排，以及原有桌面内容顺序。
- 首页工具区“精选”标题降低字号；日期标题显示为“YYYY-MM-DD 精选”。

## 实现边界

只调整 HTML 语义标记、渲染文案和 CSS 响应式布局，不改变精选数据、排序、搜索逻辑和收藏存储。

## 验证

渲染测试覆盖顶部更新时间、日期精选文案、移动搜索隐藏和底部导航样式契约；完整测试集与站点渲染必须通过。
