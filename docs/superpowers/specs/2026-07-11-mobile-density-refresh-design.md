# HN·HOT 移动端高密度布局调整设计

## 目标与范围

本次只调整 `<= 760px` 的移动端首页，不改变桌面端布局、数据结构、排序、分类、搜索、收藏存储或路由。视觉参考 AIHOT 的空间利用方式与信息密度，但保留 HN·HOT 的品牌配色、新闻内容和既有字体体系。

## 实现策略

采用同一套语义内容配合移动端媒体查询的方式实现。必要时只增加少量结构类名，不复制第二套移动端内容，也不使用 JavaScript 搬移节点。`> 760px` 的现有规则保持不变。

## 移动端页面结构

页面从上到下依次为：品牌信息、当下重点、精选标题与横向分类、按日期分组的新闻列表、固定底部导航。顶部不再占用四项主导航高度。

### 品牌区

- 顶部只保留品牌和既有日期/更新时间信息。
- 移动端顶部四项导航完全隐藏。
- 收紧品牌区下方留白，让重点内容尽早进入首屏。

### 当下重点

- 保留一个轻量面板作为重点摘要，但删除标题下横线。
- 标题字号和字重低于页面主标题。
- 面板左右内边距约 `16px`，减小上下内边距、圆角和阴影。
- 四条热点使用紧凑编号色块；标题最多两行。
- 收藏按钮缩小并固定靠右，不占用标题主要宽度。
- 条目之间仅用轻分隔或紧凑间距组织。

### 精选分类

- 移除移动端分类区域的白色外层卡片、总边框和大圆角容器。
- “精选”直接位于页面背景，使用较小章节标题。
- 分类采用单行横向滚动胶囊；隐藏滚动条但保留触摸滚动。
- 当前项使用品牌色或深色实心背景；其他项使用浅色底或细描边。
- 降低按钮字号、高度和横向内边距。

### 日期新闻列表

- 移除移动端日期分组的外层卡片、阴影和圆角。
- 日期标题直接置于页面背景。
- 列表左右页边距控制在 `16–20px`，内容尽量利用屏幕宽度。
- 每条新闻仅以浅色分隔线区分，减少上下留白。
- 标题、摘要、来源和时间沿用现有内容，但重新建立紧凑层级。
- 摘要最多三行；收藏按钮固定靠右，正文为其预留空间。

### 底部导航

- 固定四项：精选、全部信息、AI 日报、收藏。
- 图标与文字整体缩小，当前页使用品牌蓝高亮。
- 使用 `env(safe-area-inset-bottom)` 支持 iPhone 安全区域。
- 页面底部内边距等于导航实际高度加安全区，避免遮挡最后一条新闻。

## 响应式隔离

- 所有视觉覆盖限定在 `@media (max-width: 760px)`。
- 不修改桌面断点的网格、侧栏、卡片或工具区表现。
- 若必须补充 HTML 类名，新类名对桌面默认不产生布局效果。

## 可访问性与交互

- 底部导航继续使用可识别的链接文本和当前页状态。
- 收藏按钮保持可点击区域和现有可访问名称，不因图标缩小而低于合理触控面积。
- 横向分类支持键盘聚焦，焦点样式清晰。
- 标题截断只通过 CSS 行数限制，不删除完整可访问文本。

## 验证

- 渲染测试确认移动端所需结构类与底部导航仍存在。
- CSS 契约测试确认安全区、内容底部补偿、重点两行、摘要三行和横向分类滚动规则。
- 运行完整单元测试与静态站构建。
- 在接近参考图的 iPhone 视口截图检查首屏密度、导航遮挡、横向溢出和桌面端无回归。
