# 海南日报可解释编辑精选系统 V1/V2 设计

## 背景与确认现状

当前流水线已经具备可复用的基础：爬虫原始 JSON 保留版面、标题、正文和 URL；模型输入不包含 URL；输入指纹、候选 ID、模型字段和数量会被严格校验；正式日报采用临时文件替换方式写入；静态页面会转义正文内容。现有 14 个单元测试全部通过。

当前实现仍是验证模式：`prepare_model_input.py` 使用 `flatten_articles(raw)[:limit]` 只取前三篇；`finalize_digest.py` 按模型输入顺序生成排名，把分类固定为“重要但不必精读”，并生成全空分类桶；页面只呈现一个“今日看点”列表；测试集中在结构和来源注入。2026-07-08 原始样本包含 8 个版面、35 篇文章，所有文章正文非空且没有抓取错误。`hndaily-skill/_data/` 是未跟踪的用户数据，本项目只读取，不修改。

## 目标与版本边界

系统每天把整份《海南日报》转换为一份可解释、可重复的精选日报。模型只理解语义并给出五维整数评分、内容提炼和建议分类；代码独占最终分、资格、去重、选择、排名、分类合法性和每日篇数的决定权。

本轮只实施 V1：高置信硬过滤、全量候选、模型五维评分、代码最终分、海南相关性资格线、轻量事件去重、最多 8 篇、主页 Top 4、其余值得看和真实分类。V1 不强制补足 5 篇。

V2 暂缓：会议套话的复合过滤、复杂事件聚类、分类配额、七天完整 gold 指标、全参数配置化和全站原子发布。V1 不重写抓取器，不引入向量数据库，不搜索外部资料，不建设人工标注后台，不部署、推送或创建 PR。周报契约和页面保持兼容，只修改日报所需部分。所有新增实现只使用 Python 标准库。

## 方案比较与选择

### 方案 A：单次全量模型批次，加确定性后处理（采用）

代码先处理整份报纸并记录过滤结果，把全部合格候选一次性交给模型。模型返回严格的语义字段和五维评分，代码再评分、聚类、精选和编排。该方案沿用现有交接机制，调用次数少，中间产物清晰，最适合 35 篇左右的日报规模。

### 方案 B：分版面调用模型后汇总

可以降低单次上下文长度，但会增加批次一致性、部分失败、跨批次指纹和重试复杂度。第一版没有必要。

### 方案 C：本地规则完成全部分类与排名

结果最确定、成本最低，但中文新闻的行动价值、新颖性和信息密度很难靠规则可靠判断，也违背模型负责语义理解的职责边界。

## V1 文件结构与职责

```text
scripts/editorial_filter.py
  展平、规范化、长度标注、高置信硬过滤、skip_reason

scripts/editorial_scoring.py
  五维校验、基础分、确定性调整、全国转载保护、分数解释

scripts/event_clustering.py
  规范化标题和正文开头的轻量近重复判断、稳定去重、主条目选择、来源合并

scripts/select_digest.py
  海南相关性资格线、最终分阈值、最多 8 篇、稳定排名、Top 4 切分

scripts/prepare_model_input.py
  校验原始报纸，评估全部文章，原子写入预过滤审计和全量候选模型输入

scripts/finalize_digest.py
  严格校验模型输出，编排评分/聚类/精选，原子写入完整审计与正式日报

tests/
  每个 V1 模块的独立测试、流水线集成测试、真实样本回归测试和渲染测试
```

模块之间只传普通字典和列表；各核心模块不读写文件，使规则测试不依赖 CLI。CLI 层统一负责读取、严格校验和原子写入。

## V1 常量与 V2 配置化

V1 不建设完整配置系统。所有可调常量分别集中在职责模块顶部且只定义一次，禁止在 CLI、模板或多个函数中复制。V2 再迁移到经过 schema 校验的单一配置文件。V1 常量如下：

- 语义权重：`hainan_relevance=0.30`、`actionability=0.25`、`impact_scope=0.20`、`novelty=0.15`、`information_density=0.10`。
- 版面调整：头版 `+4`，本省新闻及其子版 `+3`。
- 长度调整：少于 200 字 `-8`，200–399 字 `-4`，400 字及以上 `0`。
- 精选：海南相关性资格线 `6`，最终分阈值 `65`，最多 `8`，主页重点数 `4`；不设置最少篇数。
- 去重：规范化标题完全相同直接合并；否则标题或正文开头的字符 n-gram 相似度达到模块内保守阈值才合并。
- 过滤：排除版面和精确标题集合集中在 `editorial_filter.py`。

评分模块导入时校验五维权重合计为 1。V1 代码不提供 CLI 覆盖参数，保证相同版本与相同输入得到相同结果。

## 数据流与中间契约

### 1. 原始抓取到预过滤审计

原始文章按版面号、版内 `seq` 展平，每篇获得稳定 `candidate_id`（`A001` 起）和以下元数据：原始标题、版面号、版面名、URL、作者、原始位置、正文字符数、长度等级、抓取错误。

每篇都产生过滤记录，字段至少包括 `candidate_id`、`passed`、`skip_reason`、`matched_rules` 和原始元数据。即使被跳过也保留记录，因此 2026-07-08 的 35 篇都会进入过滤评估，且数量守恒。

规则顺序固定：

1. 抓取错误或正文为空：`fetch_error` 或 `empty_content`。
2. 标题规范化后精确等于“导读”：`guide`。
3. 版面名包含“公益广告”：`public_service_ad_page`。
4. 版面名以“理论周刊”开头：`theory_weekly`。
仪式、会见、调研、表彰、开幕、迎新等内容在 V1 不做关键词硬过滤，避免误杀；它们进入模型后由行动价值、影响范围、新颖性和信息密度降低最终分。会议套话的复合过滤进入 V2。短讯不排除，只标记 `under_200` 或 `200_to_399`。

预过滤审计原子写入 `data/intermediate/YYYY-MM-DD.prefilter.json`。只把 `passed=true` 的全部候选写入模型输入；不再接受 `limit`，也不再截断前三篇。

### 2. 模型输入与输出

模型输入保持来源隔离，只含 `candidate_id`、`original_title`、`content`、`length_band` 和可选的规范化版面类型，不包含 URL、来源对象或最终规则参数。指纹覆盖 schema、prompt 版本和全部候选内容。

模型必须按输入顺序为每个候选返回且只返回：

- `candidate_id`
- `title`
- `summary`
- `why_it_matters`
- `key_facts`
- `confidence`
- `suggested_category`
- `hainan_relevance`
- `actionability`
- `impact_scope`
- `novelty`
- `information_density`
- `score_reasons`：键严格对应五个维度，每项为正文证据或简短理由

五个分数必须是 JSON 整数且位于 0–10。建议分类必须属于除“已跳过”外的固定分类集合；未知分类直接失败，不静默创造新分类。模型输出不得含 `final_score`、`selected`、`rank`、URL 或来源字段。schema、prompt 版本、输入指纹、数量、ID 集合和顺序任一不匹配都使流水线失败。

模型可在一次响应中同时产生分类建议和五维评分，但合并代码先把合法分类固化为候选阶段输出，再把评分交给评分模块；两个逻辑阶段有独立验证函数和测试。

### 3. 确定性评分

基础分使用十进制定点计算并保留一位小数：

```text
base_score = 10 × (
  relevance × 0.30 + actionability × 0.25 + impact_scope × 0.20
  + novelty × 0.15 + information_density × 0.10
)
```

最终分为基础分加有名字的调整项，并限制在 0–100。每篇保存 `semantic_scores`、`base_score`、`adjustments` 和 `final_score`。`adjustments` 是含 `rule`、`points`、`reason` 的数组，零分的适用规则也可记录。

明确为全国转载且正文与标题没有海南直接关系时，记录 `national_reprint_without_hainan_link`，抑制头版和本地版等所有正向版面加分；它仍可因模型给出的高海南相关性和其他语义分进入后续阶段，但绝不能只靠头版加权进入精选。

### 4. V1 轻量事件去重

V1 使用标准库实现的保守、确定性近重复合并：

- 标题移除空白和标点后，完全相同则合并。
- 非完全相同标题计算中文二元字符 Jaccard；正文只取开头固定长度计算三元字符 Jaccard。任一相似度达到保守阈值才合并。
- 候选按原始位置排序，只与现有事件主条目比较；低于阈值就新建事件。相同输入的事件 ID 始终一致。

主条目按以下顺序选择：正文无错误、信息密度更高、正文更长、最终分更高、版面号更小、`seq` 更小、候选 ID 更小。簇保存全部原始来源；展示使用主条目的语义内容。每个簇只进入一次最终排名。

### 5. V1 确定性精选与排序

事件先按以下稳定键排序：最终分降序、信息密度降序、正文长度降序、版面号升序、`seq` 升序、候选 ID 升序。

只有同时满足 `hainan_relevance >= 6` 和 `final_score >= 65` 的事件具备入选资格。按稳定顺序直接取前 8 个，不设置分类配额、不设置重大事件突破规则、不补入低于阈值的项目。符合条件的事件少于 8 个就如实输出相应数量，可以为 0。

未选事件保存明确原因：`below_hainan_relevance`、`below_final_score`、`daily_limit` 或 `duplicate_event`。入选事件依排序获得连续 `rank`。前 4 名就是 `top_items`，第 5–8 名是 `more_items`；不单独调用模型，不另建一套排名。若合格事件少于 4 个，主页如实展示更少。

## 正式日报契约与页面

日报保留现有顶层元数据，新增：

- `top_items`：最终精选前 4 名。
- `more_items`：最终精选第 5–8 名。
- `selected_count`、`selection_threshold`、`hainan_relevance_threshold`、`ranking_version`。
- `categories`：所有入选项目按真实分类写入；“已跳过”包含预过滤跳过记录和 `skip_reason`。

每个入选事件包含分数摘要、事件 ID、主候选 ID、全部来源和可读分数解释。完整的逐文章与逐事件审计保存在 `data/intermediate/YYYY-MM-DD.editorial-audit.json`，包括过滤、模型分、调整项、去重、选择状态、未选原因和排名。

主页沿用现有导航、归档、卡片和分类样式，只把“今日看点”改为固定的“今日重点”，渲染 `top_items`；其下新增“今日还值得看”，渲染 `more_items`。分类区域继续展示真实分类桶。全站原子发布进入 V2；V1 保持现有站点写入方式，只要求新增和修改的 JSON 文件继续使用原子替换。

## 原子性与失败处理

V1 新增和修改的 JSON 文件通过同目录临时文件写入后 `replace`。正式日报与完整审计都在内存中构建并验证完成后才落盘；先替换审计，最后替换正式日报，使正式日报成为成功发布标志。站点目录级原子替换属于 V2。

模型输出缺项、越界、类型错误、分类非法、指纹不符或候选不完整时，`finalize_digest.py` 返回非零，不覆盖既有日报，不触发渲染。流水线只负责准备交接路径，不伪造模型结果；模型输出存在并通过 finalize、validate、render 全链路后才算成功。

## V2：七天 gold 与完整回测

Gold 文件按出版日保存，包含版本、审核状态、当天全部文章或人工事件、排除原因、人工入选 5–8 项、Top 4 顺序、期望分类和可选重要性。加载器只接受 `status=approved` 的文件作为 gold；`status=suggested` 永远不能计入正式指标。

该部分不在 V1 实施。V2 会在只有 2026-07-08 原始数据的情况下，用现有爬虫抓取最近 7 个可用出版日，原始数据保留在忽略目录。工具为每一天生成：

- `evaluation/suggestions/YYYY-MM-DD.json`：明确标记为建议。
- `evaluation/review/YYYY-MM-DD.md`：全部文章/事件、建议跳过原因、建议精选和建议 Top 4。
- `evaluation/review/README.md`：一次性人工审核入口和填写说明。

`evaluate_digest.py` 用一个命令加载 7 天原始数据、系统结果和已批准 gold，输出：入选 Precision/Recall、Top 4 命中率、Top 4 顺序差异（按共同项目的归一化位置偏差）、分类准确率、错误过滤、错误入选和逐日明细。若批准的 gold 不足 7 天，校验命令明确失败；建议报告生成命令仍可运行。

## V1 测试策略

实现严格按红—绿—重构推进：

1. 过滤单测覆盖 35 篇数量守恒、导读、公益广告、理论周刊、空正文、抓取失败和短讯；会议类不因标题关键词被 V1 硬删。
2. 模型输入单测证明 2026-07-08 所有通过过滤的候选都会进入输入，公积金文章不因原始顺序被截断。
3. 模型输出单测覆盖精确字段、五维整数范围、候选 ID、数量、顺序、指纹、非法分类以及模型越权字段。
4. 评分单测用手算例验证基础分、头版/本地版、长度惩罚、全国转载保护和解释等式。
5. 去重单测覆盖标题完全相同、头版短讯与内页近重复、近似但不同事件不合并、来源完整和主条目选择。
6. 精选单测覆盖海南相关性资格线、最终分阈值、8 篇上限、稳定 tie-break、不补齐和少于 4 个合格事件。
7. 契约与渲染测试覆盖 Top 4、其余 1–4 篇、真实分类桶、HTML 转义和原子写入。
8. 集成测试运行准备、合并、校验和渲染；真实样本回归证明 35 篇先被评估且公积金文章进入候选。

## V1 实施阶段

1. 集中常量、预过滤和全量模型输入。
2. 新模型契约和确定性评分。
3. 轻量事件去重、主条目与来源合并。
4. 双资格线、最多 8 篇精选、真实分类和新版日报契约。
5. Top 4/其余内容静态页面编排与原子渲染。
6. 文档更新、完整测试、2026-07-08 真实样本生成与页面验证。

每一阶段都有独立失败测试和可运行产物；不在阶段之间隐藏模型决策或使用未记录的排序规则。

## 验收映射

- 三篇截断由“35 篇全部过滤评估、全部合格候选进模型输入”的集成测试替代。
- 跳过项拥有稳定 `skip_reason`；短讯只降权。
- 模型只有五维语义分，代码独占最终分和排名。
- 权重和阈值在各自职责模块中集中定义且不重复；V2 再迁移到单一配置文件。
- 每个事件只占一个名额并保留全部来源。
- 每天最多 8 篇，不强制补足；前 4 和第 5–8 篇来自同一排名。
- 分类桶包含真实文章，未知分类使流程失败。
- 原有测试在契约迁移后继续覆盖原意，新增质量规则都有行为测试。
- 真实 2026-07-08 样本完成 prepare、finalize、validate、render 后才视为验收通过。

## V2 暂缓清单

- 会议套话的“噪声语境 + 无具体事实”复合过滤。
- 日期、机构、地点、数字和头版/内页关系参与的复杂聚类。
- 分类配额、重大事件突破和最低篇数补齐策略。
- 七天人工 gold、完整 Precision/Recall、Top 4 排序差异和分类准确率回测。
- 经 schema 校验的全参数配置文件。
- 站点目录级原子发布和失败回滚。
