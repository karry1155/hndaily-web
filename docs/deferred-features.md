# 延后开发功能清单

## 文档用途

本文件记录核心中文深色版本稳定后再着手设计、开发和验收的功能。这些功能不进入当前阶段的实现范围，不创建占位页面，也不参与当前阶段的截图验收。

## 1. 英文版

- 英文版整体延后开发。
- 当前阶段只生产和验收中文版。
- 影响范围使用稳定的内部枚举，当前中文标签映射为：

| 内部枚举 | 中文标签 |
|---|---|
| `national` | 全国 |
| `hainan` | 海南本地 |
| `domestic` | 国内关联 |
| `mixed` | 海南开放 |
| `foreign` | 全球 |

- 英文界面文案、新闻标题与正文是否翻译、AI 摘要翻译方式、英文搜索索引和中英文切换方式，均在英文版启动时另行设计，不提前混入当前数据契约。

## 2. 浅色版本与跟随系统

- 当前阶段只开发和验收深色版本。
- 浅色版本延后开发。
- 跟随操作系统自动切换深浅主题的功能延后开发。
- 当前阶段不制作浅色截图，不进行深浅两套重复视觉验收。
- 浅色版本启动时，再统一补充颜色令牌、系统主题监听、用户选择持久化和双主题回归测试。

## 启动条件

只有在以下核心能力完成并通过验收后，才重新评估本清单中的功能：

1. 原始报纸到唯一标准报道的数据生产链稳定。
2. 头版、全部、日报和更多四个核心区域完成。
3. 主体、地区与连续报道关系具备可核验的真实数据。
4. 中文深色版本完成桌面端和移动端验收。
